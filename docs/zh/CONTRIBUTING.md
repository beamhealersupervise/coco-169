## 🚀 开始贡献

以下是开始贡献的步骤：

1. 🍴 **Fork 仓库**：将 Coco Fork 到你的 GitHub 账户。

2. 🛠️ **克隆你的 Fork**：
   ```bash
   git clone https://github.com/YOUR_USERNAME/coco.git
   cd coco
   ```

3. 🔧 **设置开发环境**：
   ```bash
   # 安装 Go 依赖
   go mod tidy
   
   # 设置前端（如果需要修改 UI）
   cd frontend
   npm i
   ```

4. 🌱 **创建分支**：
   ```bash
   git checkout -b feature/你的功能名称
   ```

5. 🚀 **提交 Pull Request**：准备好后，向主仓库提交 PR！

## 🎯 开发工作流

### 后端（Go）

1. **编写代码**：遵循 Go 最佳实践和约定
2. **格式化代码**：提交前运行 `gofmt -s -w .`
3. **运行测试**：确保所有测试通过
   ```bash
   go test ./...
   go test -race ./...  # 检查竞态条件
   ```
4. **构建**：验证构建是否正常
   ```bash
   go build
   ```

### 前端（Vue 3）

1. **开发服务器**：运行开发服务器
   ```bash
   cd frontend
   npm run dev
   ```
2. **格式化代码**：使用 Prettier 格式化
   ```bash
   npm run format
   ```
3. **类型检查**：确保 TypeScript 类型正确
   ```bash
   npm run type-check
   ```
4. **构建**：测试生产构建
   ```bash
   npm run build
   ```

## 🌠 创建 Pull Request

提交 PR 时：

### ✅ 检查清单

- [ ] 代码已格式化（Go 用 `gofmt`，前端用 `prettier`）
- [ ] 所有测试通过
- [ ] 新功能有测试
- [ ] 文档已更新（如需要）
- [ ] 提交信息清晰且描述性强
- [ ] PR 描述解释了做了什么以及为什么

### 📝 PR 指南

- **标题**：使用清晰、描述性的标题
  - ✅ `feat: 添加 OpenAPI 3.1 支持`
  - ✅ `fix: 修复主题切换 bug`
  - ✅ `docs: 更新安装指南`
  
- **描述**：解释：
  - 你做了什么修改
  - 为什么做这些修改
  - 如何测试
  - 是否有破坏性变更

- **大小**：保持 PR 专注且大小合理
  - 大的改动？考虑拆分成多个小 PR
  - 每个 PR 应该解决一个问题

## 👁️ 代码审查

所有 PR 都会经过代码审查。我们要关注：

### 🧙‍♀️ 代码质量

- 遵循 [Go Code Review Comments](https://github.com/golang/go/wiki/CodeReviewComments)
- 编写清晰、自文档化的代码
- 为复杂逻辑添加注释
- 正确处理错误
- 避免不必要的复杂性

### 📝 提交信息

编写有意义的提交信息：

```
feat: 为 API 浏览器添加暗色模式支持

- 在头部实现主题切换
- 为主题颜色添加 CSS 变量
- 在 localStorage 中持久化主题偏好

Closes #123
```

遵循以下格式：
- **类型**：`feat`、`fix`、`docs`、`style`、`refactor`、`test`、`chore`
- **主题**：简短描述（50 个字符或更少）
- **正文**：详细说明（如需要）
- **页脚**：引用 issue/PR

### 🔍 审查流程

1. 维护者会审查你的 PR
2. 处理任何反馈或请求的更改
3. 一旦批准，你的 PR 将被合并！

```

├── internal/            # 内部包
│   ├── handler.go       # HTTP 处理逻辑
│   ├── config/          # 配置
│   └── assets/          # 嵌入的前端资源
├── frontend/            # Vue 3 前端
│   ├── src/             # 源代码
│   ├── vite.config.ts   # 构建配置
│   └── package.json     # 依赖
├── example/             # 示例集成
```
